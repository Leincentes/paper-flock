# Paper Flock v1.4.2 — Startup Recovery Hotfix Report

## Decision

**IMPLEMENTED AND READY FOR GITHUB-HOSTED RELEASE QUALIFICATION**

v1.4.2 fixes the startup-recovery crash reported in v1.4.1 without changing
gameplay rules, achievements, progression, storage keys, or save schema 12.

## Reported failure

```text
Uncaught TypeError: Cannot set properties of null (setting 'textContent')
```

The failure appeared after the tutorial and checkpoint had already been saved.
The supplied primary save and backup were structurally consistent with schema
12. The crash was not caused by corrupted player data.

## Root cause

`src/game-player-ui.js` queried:

```js
masteryGoal: document.querySelector("#mastery-goal")
```

and then assigned `elements.masteryGoal.textContent` during `updateHud()`.
The v1.4.1 production `index.html` did not contain an element with the
`mastery-goal` ID. Therefore the query returned `null`, and the first HUD
update entered startup recovery.

The mismatch escaped v1.4.1 because tests verified that the Journal modules
were shipped but did not verify the complete runtime-to-HTML selector contract.

## Fix

1. Added the required `#mastery-goal` element to the player HUD.
2. Added defensive checks before mastery-goal text updates.
3. Added a test that compares required `game-player-ui.js` ID selectors with
   production HTML.
4. Added a focused regression test for the mastery-goal element and guard.
5. Advanced the release and service-worker cache version to v1.4.2.

## Save compatibility

- Stable storage keys are unchanged.
- Save schema remains **12**.
- Existing v1.4.1 saves and backups remain compatible.
- No progress reset is required.
- No migration is required.
- The reported checkpoint can be restored by the corrected runtime.
