# Paper Flock v0.11 — Full Field-Test Implementation Report

**Build:** v0.11  
**Decision:** **READY TO COLLECT 8–10 REAL FIELD SESSIONS**

## 1. Purpose

v0.11 executes the next recommendation without fabricating customer evidence.
It turns the real-device core-feel test into a self-guided field study that can
run on participants' own phones and combine returned anonymous session files on
the creator's device.

## 2. Packaging defects corrected

The v0.10 package contained the tactile test source code but did not load
`src/tactile-test-ui.js` from `index.html`. As a result, the documented
`tactiletest=1` mode could not initialize.

The customer interface also referenced an Effects control in JavaScript without
providing the corresponding `effects-button` element.

v0.11 fixes both defects:

- tactile and field-test UI module is loaded
- Sound and Effects controls are both present and labeled
- existing tactile experience test now passes
- game DOM references are complete

## 3. Self-guided field-test flow

Open:

```text
?fieldtest=1
```

The participant completes:

1. Anonymous consent and broad segment selection
2. Device-tier selection
3. Clean natural play with Sound off and Effects Auto
4. Guided sound and haptic comparison
5. Lite-effects comparison on low-end devices
6. Final tactile and accessibility survey
7. Anonymous JSON download

The coach uses small bottom prompts and does not explain blocked-path or
rotation rules before the participant encounters them.

## 4. Remote evidence collection

Each participant can return one JSON file. The creator can import one or many
files through **Prototype testing tools**.

Import behavior:

- accepts single-session and combined-session exports
- rejects malformed session records
- skips duplicate session IDs
- skips duplicate anonymous participant codes
- preserves valid previously imported sessions
- updates recruitment and protocol coverage immediately

Nothing is transmitted automatically.

## 5. Field coverage gate

A product decision requires both strong product metrics and a sufficiently
representative sample.

Minimum coverage:

| Requirement | Minimum |
|---|---:|
| Protocol-complete sessions | 8 |
| Casual puzzle players | 4 |
| Experienced puzzle players | 1 |
| Casual non-puzzle players | 1 |
| Accessibility or sensitivity perspective | 1 |
| Low-end or older phone | 1 |
| Sound-tested sessions | 3 |
| Haptic-tested compatible sessions | 3 |
| Blocked-path eligible sessions | 3 |
| Rotation-learning eligible sessions | 3 |

The dashboard recommends which participant perspective to recruit next.

## 6. Protocol-complete session rules

A session counts toward the field gate when it contains:

- completed final survey
- at least three tap-latency samples
- at least sixty frame samples
- at least two minutes between start and end
- sound evaluation when Web Audio is supported
- haptic evaluation when vibration is supported
- Lite or Minimal effects observed on low-end sessions

Incomplete sessions remain visible and importable but do not satisfy the final
coverage requirement.

## 7. Product acceptance criteria

| Measure | Target |
|---|---:|
| Tap response perceived as immediate | ≥ 90% |
| Blocked path correctly explained | ≥ 85% |
| Rotations understood by Level 3 | ≥ 80% |
| Movement feedback rated 4/5+ | ≥ 75% |
| Tested sound pleasant or neutral | ≥ 70% |
| Tested haptics pleasant or neutral | ≥ 70% |
| Effects distracting | < 15% |
| Repeated low-end stutter | 0 |
| Critical blockers | 0 |

## 8. Automated decision options

- **READY FOR CLOSED ALPHA**
- **READY FOR ANOTHER CORE-FEEL ROUND**
- **REDESIGN BLOCKED-PATH FEEDBACK**
- **REDESIGN SOUND OR HAPTICS**
- **REDUCE VISUAL EFFECTS**

A strong convenience sample cannot pass if the required player and device
perspectives are missing.

## 11. Privacy and ethics

- no names, email addresses, phone numbers, or locations requested
- no automatic upload
- no account or advertising identifier
- explicit consent
- participant may stop
- free-text exports must be reviewed for accidental personal information
- minors should not be recruited without appropriate consent and policy review

## 12. Limitations

- No real participants were available in this environment.
- No real-phone sound, haptic, touch, or low-end performance result is claimed.
- The execution environment did not permit a successful bound HTTP smoke test.
- Automated timing and synthetic records cannot establish perceived quality.
- Closed alpha remains blocked until the real evidence files are imported.

## 13. Final audit

**PASS WITH CONDITIONS**

Conditions:

1. Collect 8–10 real sessions.
2. Include the required low-end and accessibility perspectives.
3. Import the returned JSON files.
4. Review observed behavior before free-text opinions.
5. Do not add monetization or new retention systems before the gate.
6. Advance only when the combined field evaluation supports it.
