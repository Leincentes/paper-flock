# Paper Flock v1.6.0 Mobile “Choose a Flock” Hotfix Report

## Status

**Local hotfix candidate — not production-approved.**

## Reported symptom

On mobile layouts, the game remains on the **Choose a flock** level map after the player selects an unlocked level. The puzzle is initialized underneath the map, but the map remains visible and blocks interaction.

## Root cause

The base stylesheet correctly defines:

```css
.level-map-overlay[hidden] {
  display: none;
}
```

A later mobile containment rule defines:

```css
html.mobile-gameplay-lock .level-map-overlay {
  display: flex;
}
```

The later mobile selector has greater CSS specificity, so it overrides the `hidden` state on phone layouts. JavaScript correctly sets `levelMap.hidden = true` after selection, but the CSS keeps rendering the overlay.

## Fix

A higher-specificity hidden-state guard now follows the mobile containment rule:

```css
html.mobile-gameplay-lock .level-map-overlay[hidden] {
  display: none;
}
```

This preserves the full-height, scroll-contained mobile map while allowing normal JavaScript navigation to close it.

The service-worker cache identifier was changed to:

```text
paper-flock-static-v1.6.0-mobile-map-hotfix-1
```

This prevents previously installed PWAs from indefinitely reusing the defective stylesheet.

## Regression coverage

- Extended the mobile UI audit to reject builds without the hidden-state guard.
- Extended the configured Playwright mobile flow to select an unlocked flock and verify that the level map becomes hidden and the board remains visible.
- Synchronized the corrected production files into the Android WebView assets.

## Remaining validation

The configured browser test was not executed in this environment because the project dependencies and hosted browser runners are not available here. Android lint, AAB assembly, physical Android Chrome/WebView testing, PWA upgrade testing, and real-player testing remain required.

Recommended physical-device acceptance check:

1. Install or refresh the candidate.
2. Open **More → Level map**.
3. Tap any unlocked level.
4. Confirm the map closes immediately and the selected puzzle accepts input.
5. Repeat in portrait, landscape, browser-tab, installed-PWA, and Play wrapper modes.

## Release decision

The root cause is identified and patched in the source, production distribution, and Android assets. The candidate is suitable for controlled device testing, but production approval remains blocked on hosted and physical-device evidence.
