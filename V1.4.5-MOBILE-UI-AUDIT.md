# Paper Flock v1.4.5 — Mobile UI Audit and Repair Report

## Decision

**IMPLEMENTED — READY FOR GITHUB-HOSTED BROWSER AND ANDROID QUALIFICATION**

The mobile player interface was rebuilt around a board-first layout and audited across portrait phones, short phones, compact landscape, utility dialogs, the level map, the tutorial, Settings, the Achievement Journal, and all production controls.

## Scope

The repair covers:

- gameplay HUD and puzzle board
- Undo, Hint, Restart, and More
- level navigation
- level map
- Daily Flock
- Settings
- Achievement Journal
- tutorial
- completion and confirmation dialogs
- public information links
- safe-area, short-viewport, and compact-landscape behavior
- Android WebView asset synchronization

Gameplay rules, save schema 12, achievements, level data, and existing player preferences remain unchanged.

## Implemented changes

### Mobile gameplay

- Replaced the fragile implicit mobile rows with named CSS grid areas.
- Made the puzzle board the flexible primary region.
- Consolidated the four primary actions into one stable bottom bar:
  - Undo
  - Hint
  - Restart
  - More
- Raised primary mobile controls to a minimum height of 48 CSS pixels.
- Kept critical-height controls at or above 44 CSS pixels.
- Added viewport-aware board sizing using the visual viewport.

### More menu

- Grouped secondary features into four labelled sections:
  - Level navigation
  - Player tools
  - Explore
  - Information
- Ensured the drawer closes before opening Settings, Journal, level map, or Daily Flock.
- Added modal semantics, backdrop dismissal, Escape handling, focus trapping, and focus restoration through the central accessibility controller.
- Made the menu internally scrollable while the gameplay page remains locked.

### Dialogs and utility surfaces

- Kept Settings, Journal, tutorial, level map, Daily Flock, and completion surfaces within the visible mobile viewport.
- Added internal scrolling and overscroll containment.
- Added sticky level-map header and close control.
- Increased interactive targets to at least 44 CSS pixels for coarse-pointer/mobile layouts.
- Reduced nonessential text only on extremely narrow screens without hiding core actions.

### Landscape

- Added a compact landscape layout with:
  - board on the left
  - HUD, instruction, and a two-by-two action rail on the right
- Prevented controls from stretching vertically.
- Preserved a board target of at least 280 CSS pixels in the configured browser regression.

### Android

- Advanced Android versionName to 1.4.5 and versionCode to 10405.
- Synchronized the final production web assets into the local Android WebView bundle.
- Preserved target SDK 36, minimum SDK 24, local-only operation, and the single VIBRATE permission.

## Browser regressions added

The Playwright suite now checks:

- no horizontal overflow
- visible-viewport page lock
- short-phone board and control reachability
- internal More and Settings scrolling
- one-row four-action mobile control bar
- 44-pixel utility touch targets
- More-menu dismissal before opening another surface
- level-map viewport containment
- compact-landscape board and control dimensions
- tutorial viewport containment
- Journal internal scrolling

A local system-Chromium attempt could not complete in this container because Chromium's GPU process terminates before a browser context is created. This is an environment limitation, not a passing browser result. GitHub-hosted Chromium, mobile Chromium, WebKit, Android lint, and the signed AAB remain the final qualification gates.
